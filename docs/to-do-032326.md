# To-Do — 2026-03-23

Items identified during the cert renewal crash and old binary incident.

## 1. Add build version stamp to binary

Add a `-ldflags` build timestamp so the startup log identifies exactly which build is running. This helps diagnose situations where an unexpected binary version is active.

- Add `var BuildTime string` in `cmd/stratahub/main.go`
- Log it at startup (e.g., `"stratahub starting", zap.String("build", BuildTime)`)
- Update `Makefile` build-linux target to include:
  ```
  -ldflags "-X main.BuildTime=$(date -u +%Y%m%d-%H%M%S)"
  ```
- Optionally expose via a `/version` endpoint for quick checks

## 2. Clean up stale .new files before deploy

Add an early step to `aws_update.sh` to remove any leftover `.new` files before uploading:

```bash
ssh ... "rm -f /home/ubuntu/stratahub-linux-386.new /home/ubuntu/config.toml.new"
```

Prevents any edge case where a stale `.new` file from a failed prior deploy gets swapped in.

## 3. Add AWS_REGION to the main systemd service file

Currently added via `override.conf` (`systemctl edit`). Move it into the main service file so it survives instance replacements or service file reinstalls:

```ini
Environment=AWS_REGION=us-east-2
```

Add below the existing `Environment=STRATAHUB_CONFIG_PATH=...` line in `/etc/systemd/system/stratahub.service`.
